Qual a finalidade das migrações?

Modificar a estrutura do banco de dados, exemplo: criar tabela, eliminar tabela,
modificar tabela, configurar restrições (constraint: primary key, foreign key,
índice, unique, check).

Quais funções existem nas migrações e para que servem:

UP - faz alteração estrutural no BD
DOWN - desfaz a alteração.

-----------------

Qual significado dos códigos de status:

2xx - sucesso, operação realizada  

4xx - erro devido a dados inválidos enviados pelo cliente

5xx - erro interno a aplicação

Dado o componente react abaixo, quantas vezes ele é 
renderizado (redesenhado - seu código executado)?

export default function Teste()
{
  const g = 0
  const [numero,setNumero] = useState(0)
  return(
     <h1>......
     ........
  )
}

Para (qual a finalidade) que usar um useState?

Armazenamento de dados. Preserva os dados enquanto o componente
permanecer montado.

A instrução abaixo é correta, considerando o useState acima?
	numero = numero + 1
Resp.: Incorreta, não é possível alterar uma constante

Como fazer então?

	setNumero(numero+1)

---------------

Para que serve useRef?

Por exemplo, para armazenar os dados digitados pelo usuário
em um formulário. ARMAZENAMENTO DE DADOS

Qual é a diferença principal entre useState e useRef?

Mudanças no valor do useState causam nova renderização do componente

Mudanças no useRef não causam renderização.
